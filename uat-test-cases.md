# UAT 用户验收测试用例

> 测试环境：飞书私聊 / 群聊 + Web 端
> 前置条件：Bot 已部署，Web 已构建，飞书登录可用

---

## 1. 项目创建 — 发起创建

Bot 收到创建意图后，检查必填字段（名称、品类、部门、预算、计划开始/结束日期），缺哪个追问哪个。

| # | 输入对话 | 预期行为 | 通过 |
|---|---------|---------|------|
| 1.1 | 创建一个新项目 | 追问缺少的信息 | |
| 1.2 | 新建项目 | 同上 | |
| 1.3 | 帮我建个项目 | 同上 | |
| 1.4 | 我要开一个新的采购项目 | 同上 | |
| 1.5 | 搞个项目 | 同上 | |
| 1.6 | 添加一个项目 | 同上 | |
| 1.7 | 创建项目ABC | 追问品类、部门、预算、日期（名称已填） | |
| 1.8 | 创建项目ABC，品类设备 | 追问部门、预算、日期（名称+品类已填） | |
| 1.9 | 创建项目ABC，品类设备，部门FBU，预算100万，1月到12月 | 信息完整 → 发送确认卡片 | |
| 1.10 | 创建一个设备采购项目，叫测试项目，FBU部门，预算200万，明年全年 | 信息完整 → 发送确认卡片 | |
| 1.11 | 创建项目ABC（名称已存在） | 提示"项目名称已存在（编号：CG-XXX），请换个名称"，不继续追问 | |

## 4. 项目创建 — 确认与取消

信息完整后 Bot 发送确认卡片，用户通过文字或按钮确认/取消。

| # | 输入对话 / 操作 | 预期行为 | 通过 |
|---|---------|---------|------|
| 4.1 | 确认 | 触发创建 | |
| 4.2 | 确定 | 触发创建 | |
| 4.3 | 好的 | 触发创建 | |
| 4.4 | 可以 | 触发创建 | |
| 4.5 | 是的 | 触发创建 | |
| 4.6 | 是 | 触发创建（≤2字符） | |
| 4.7 | 好 | 触发创建（≤2字符） | |
| 4.8 | 行 | 触发创建（≤2字符） | |
| 4.9 | 没问题 | 触发创建 | |
| 4.10 | 点击卡片"确认创建"按钮 | 触发创建，返回项目详情 | |
| 4.11 | 是一个好项目 | **不**触发确认（>2字符，视为普通对话） | |
| 4.12 | 可以吧 | **不**触发确认（>2字符） | |
| 4.13 | 取消 | 取消创建 | |
| 4.14 | 不要 | 取消创建 | |
| 4.15 | 算了 | 取消创建 | |
| 4.16 | 不做了 | 取消创建 | |
| 4.17 | 否 | 取消创建（≤2字符） | |
| 4.18 | 停 | **不**触发取消（不在取消词列表） | |
| 4.19 | 点击卡片"取消"按钮 | 取消创建 | |

## 14. 错误处理

| # | 输入对话 / 操作 | 预期行为 | 通过 |
|---|---------|---------|------|
| 14.1 | 创建同名项目（输入名称时即校验） | 提示"项目名称已存在（编号：CG-XXX），请换个名称"，不继续追问其他字段 | |
| 14.2 | Bot 收到空消息 | 不崩溃，无响应 | |
| 14.3 | Bot 收到图片/文件等非文本消息 | 不崩溃，无响应 | |
| 14.4 | 群聊中非@消息（无活跃会话） | 静默忽略 | |
| 14.5 | 发送乱码如"asdfghjkl" | 返回默认引导提示 | |
| 14.6 | 发送纯数字如"12345" | 返回默认引导提示 | |
