# 子应用自动注册接入教程

> 版本：v2.3 | 更新时间：2026-06-27

---

## 一、概述

本文档面向生态伙伴开发者，介绍如何将你的应用改造为 HRAS 平台的子应用模块，实现**启动即自动注册**到壳子平台。

子应用与壳子的关系：

```
┌────────────────────────────────────┐
│          HRAS 壳子平台             │
│                                    │
│  ┌──────────┐    注册信息上报      │
│  │ 注册中心  │◄───────────────────┼─── 你的子应用 (启动时)
│  └──────────┘                     │
│  ┌──────────┐    健康检查/指标采集 │
│  │ 监控采集  │───────────────────►│─── 你的子应用 (定时)
│  └──────────┘                     │
│  ┌──────────┐    iframe 嵌入      │
│  │ 模块容器  │───────────────────►│─── 你的子应用前端
│  └──────────┘                     │
└────────────────────────────────────┘
```

### 接入前需知

- 壳子**不限制**你的技术栈：后端任意语言，前端任意框架
- 模块之间**不通信**，完全独立部署
- 壳子只控制"能不能看到你的模块"，模块内部权限你自己管理
- 壳子地址（本地开发默认）：`http://localhost:8066`
- 壳子前端地址（本地开发默认）：`http://localhost:3000`

## 二、改造步骤总览

| 步骤 | 内容 | 必须 |
|------|------|:--:|
| 1 | 后端启动时调用注册 API | ✅ |

| 5 | 后端验证壳子 JWT Token（可选） | 🟡 |
| 6 | 操作日志上报（可选） | 🟡 |


下面按步骤逐一说明。

## 三、步骤 1：后端启动时调用注册 API

### 3.1 注册 API

```
POST /api/modules/register
Content-Type: application/json
Authorization: Bearer {apiKey}
```

> **注意**：此接口需要携带模块 **apiKey** 进行认证。apiKey 由管理员在壳子后台「模块管理」中创建模块标识时生成，请联系管理员获取。

### 3.1.1 如何获取 apiKey

1. 联系壳子平台管理员，提供你的模块标识（`module_key`）
2. 管理员在壳子后台「模块管理」中创建模块标识，建成功后会自动生成 apiKey
3. 管理员将 apiKey 提供给你，**请妥善保管**

> ⚠️ apiKey 仅展示一次（创建成功时），关闭弹窗后无法再次查看。如需重新生成，联系管理员在模块编辑中操作。

### 3.2 请求体说明

```json
{
  "module_key": "your-app-key",       // 【必填】模块唯一标识，英文+数字+中划线
  "name": "你的模块名称",              // 【必填】在壳子菜单中显示的名称
  "frontend_url": "http://你的前端地址",
  "backend_url": "http://你的后端地址",
  "health_path": "/health",           // 健康检查路径，默认 /health
  "metrics_path": "/metrics",         // 指标路径，默认 /metrics
  "menu": {                           // 菜单配置
    "icon": "your-icon",              // Ant Design 图标名称
    "children": [
      { "path": "/page1", "name": "页面1" },
      { "path": "/page2", "name": "页面2" }
    ]
  }
}
```

### 3.3 调用时机

**后端服务启动完成后**立即调用。不同框架的钩子方式如下：

| 框架 | 推荐方式 |
|------|----------|
| Spring Boot | `ApplicationRunner` / `@PostConstruct` |
| FastAPI | `@app.on_event("startup")` / `lifespan` |
| Gin/Echo | `main()` 中 `go registerModule()` |
| Express/Koa | `app.listen()` 回调中调用 |

> **关键**：注册失败不应阻止应用启动。壳子不可达时，模块应降级为独立运行模式。

### 3.4 环境变量配置（推荐）

不要将壳子地址、模块标识硬编码在代码中，通过环境变量注入：

```bash
# 壳子后端地址（本地开发默认 http://localhost:8066）
export HRAS_SHELL_URL=http://your-shell:8066

# 是否在启动时自动注册（默认 true，设为 false 则跳过）
export HRAS_SHELL_REGISTER_ENABLED=true

# 模块唯一标识（格式：{业务域}-{模块名}）
export HRAS_MODULE_KEY=hras-your-module

# 模块在壳子菜单中显示的名称
export HRAS_MODULE_NAME=你的模块名称

# 模块前端地址（壳子用此地址生成 iframe）
export HRAS_MODULE_FRONTEND_URL=http://your-frontend:3000

# 模块后端地址
export HRAS_MODULE_BACKEND_URL=http://your-backend:8066

# 模块 apiKey（从壳子管理员处获取，用于注册认证）
export HRAS_MODULE_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> **注意**：`HRAS_SHELL_REGISTER_ENABLED` 默认为 `true`。如果启动时没有看到注册日志，检查是否被设为 `false`。

### 3.5 代码示例（Spring Boot / Java）

```java
package com.example.module;

import org.springframework.boot.ApplicationRunner;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.web.client.RestTemplate;

@SpringBootApplication
public class YourModuleApplication {

    public static void main(String[] args) {
        SpringApplication.run(YourModuleApplication.class, args);
    }

    @Bean
    public ApplicationRunner registerToShell() {
        return args -> {
            try {
                String shellUrl = "http://localhost:8066/api/modules/register";
                String apiKey = System.getenv("HRAS_MODULE_API_KEY");
                String registrationBody = """
                {
                    "module_key": "your-app-key",
                    "name": "你的模块名称",
                    "frontend_url": "http://your-frontend:3000",
                    "backend_url": "http://your-backend:8066",
                    "health_path": "/health",
                    "metrics_path": "/metrics",
                    "menu": {
                        "icon": "appstore",
                        "children": [
                            { "path": "/home", "name": "首页" },
                            { "path": "/settings", "name": "设置" }
                        ]
                    }
                }
                """;

                RestTemplate restTemplate = new RestTemplate();
                org.springframework.http.HttpHeaders headers = new org.springframework.http.HttpHeaders();
                headers.setContentType(org.springframework.http.MediaType.APPLICATION_JSON);
                headers.setBearerAuth(apiKey);
                org.springframework.http.HttpEntity<String> entity =
                    new org.springframework.http.HttpEntity<>(registrationBody, headers);
                String response = restTemplate.postForObject(shellUrl, entity, String.class);
                System.out.println("[模块注册] 注册成功: " + response);
            } catch (Exception e) {
                System.err.println("[模块注册] 注册失败: " + e.getMessage());
                // 不要因为注册失败而阻止应用启动
            }
        };
    }
}
```

### 3.6 代码示例（Go）

```go
package main

import (
    "bytes"
    "net/http"
    "os"
)

func registerModule() {
    apiKey := os.Getenv("HRAS_MODULE_API_KEY")
    body := `{
        "module_key": "your-app-key",
        "name": "你的模块名称",
        "frontend_url": "http://your-frontend:3000",
        "backend_url": "http://your-backend:8066",
        "health_path": "/health",
        "metrics_path": "/metrics",
        "menu": {
            "icon": "appstore",
            "children": [
                { "path": "/home", "name": "首页" }
            ]
        }
    }`

    req, err := http.NewRequest("POST",
        "http://localhost:8066/api/modules/register",
        bytes.NewBuffer([]byte(body)))
    if err != nil {
        println("[模块注册] 创建请求失败:", err.Error())
        return
    }
    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("Authorization", "Bearer "+apiKey)

    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        println("[模块注册] 注册失败:", err.Error())
        return
    }
    defer resp.Body.Close()
    println("[模块注册] 注册成功, status:", resp.StatusCode)
}
```

### 3.7 代码示例（Python / FastAPI，完整版）

**config.py** — 环境变量驱动的配置：

```python
import os

HRAS_SHELL_URL = os.environ.get("HRAS_SHELL_URL", "http://localhost:8066")
HRAS_SHELL_REGISTER_ENABLED = os.environ.get("HRAS_SHELL_REGISTER_ENABLED", "true").lower() == "true"
HRAS_MODULE_KEY = os.environ.get("HRAS_MODULE_KEY", "hras-your-module")
HRAS_MODULE_NAME = os.environ.get("HRAS_MODULE_NAME", "你的模块名称")
HRAS_MODULE_FRONTEND_URL = os.environ.get("HRAS_MODULE_FRONTEND_URL", "http://localhost:8000")
HRAS_MODULE_BACKEND_URL = os.environ.get("HRAS_MODULE_BACKEND_URL", "http://localhost:8000")
HRAS_MODULE_API_KEY = os.environ.get("HRAS_MODULE_API_KEY", "")
```

**app.py** — 在 `on_event("startup")` 中异步注册：

```python
import httpx
from fastapi import FastAPI

from .config import (
    HRAS_SHELL_URL, HRAS_SHELL_REGISTER_ENABLED,
    HRAS_MODULE_KEY, HRAS_MODULE_NAME,
    HRAS_MODULE_FRONTEND_URL, HRAS_MODULE_BACKEND_URL,
    HRAS_MODULE_API_KEY,
)

app = FastAPI()


def _build_registration_payload() -> dict:
    return {
        "module_key": HRAS_MODULE_KEY,
        "name": HRAS_MODULE_NAME,
        "frontend_url": HRAS_MODULE_FRONTEND_URL,
        "backend_url": HRAS_MODULE_BACKEND_URL,
        "health_path": "/health",
        "metrics_path": "/metrics",
        "menu": {
            "icon": "appstore",
            "children": [{"path": "/", "name": "首页"}],
        },
    }


@app.on_event("startup")
async def _register_to_shell():
    if not HRAS_SHELL_REGISTER_ENABLED:
        print("[模块注册] 已跳过（HRAS_SHELL_REGISTER_ENABLED=false），模块独立运行。")
        return
    data = _build_registration_payload()
    print(f"[模块注册] 正在向壳子注册: {HRAS_SHELL_URL}/api/modules/register")
    try:
        async with httpx.AsyncClient(timeout=10) as client:
            resp = await client.post(
                f"{HRAS_SHELL_URL}/api/modules/register",
                json=data,
                headers={"Authorization": f"Bearer {HRAS_MODULE_API_KEY}"},
            )
            resp.raise_for_status()
            print(f"[模块注册] ✅ 注册成功 | module_key={HRAS_MODULE_KEY} | status={resp.status_code}")
    except Exception as exc:
        print(f"[模块注册] ⚠️ 注册失败（壳子可能未启动），模块仍可独立运行。错误：{exc}")
        # 不阻止应用启动
```

**依赖清单**（requirements.txt）：

```
fastapi
uvicorn
httpx
```

> **重要**：
> - FastAPI 中必须使用 **`httpx`（异步）** 而非 `requests`（同步），否则启动事件会阻塞
> - 使用 `print()` 而非 `logging` 可将注册日志直接输出到终端，便于排查
> - `HRAS_SHELL_REGISTER_ENABLED` 默认为 `true`，**确保该开关未被关闭**

### 3.8 module_key 命名规范

- 使用**小写英文字母 + 数字 + 中划线**
- 格式：`{业务域}-{模块名}`，如 `hras-attendance`、`pdf-item`
- 长度不超过 64 字符
- 全局唯一，注册后不可修改

## 四、步骤 2：提供 `/health` 健康检查端点

### 4.1 要求

壳子会每隔 **30 秒** 轮询一次该接口，用于监控看板展示模块状态。

### 4.2 返回格式

```json
{
  "status": "UP"
}
```

- `"UP"` 表示正常
- 其他任意值或不可达视为异常

### 4.3 Spring Boot 实现（推荐用 Actuator）

如果你用 Spring Boot，只需添加 Actuator 依赖即可自动获得 `/actuator/health` 端点。然后将 `health_path` 配置为 `/actuator/health`。

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

### 4.4 手写实现

```java
@RestController
public class HealthController {

    @GetMapping("/health")
    public Map<String, String> health() {
        return Map.of("status", "UP");
    }
}
```

Go:
```go
http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
    w.Header().Set("Content-Type", "application/json")
    w.Write([]byte(`{"status":"UP"}`))
})
```

Python / FastAPI:
```python
@app.get("/health")
def health():
    return {"status": "UP"}
```

## 五、步骤 3：提供 `/metrics` 监控指标端点

### 5.1 要求

壳子每 30 秒拉取一次，用于监控看板展示指标趋势。

### 5.2 返回格式

```json
{
  "status": "UP",
  "timestamp": 1716854400,
  "base": {
    "request_count": 12345,
    "error_rate": 0.02,
    "avg_response_ms": 120,
    "cpu_percent": 45.2,
    "memory_mb": 512
  },
  "custom": {
    "today_attendance_count": 230,
    "pending_approval": 15
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | String | `"UP"` 或 `"DOWN"` |
| `timestamp` | Number | Unix 秒级时间戳 |
| `base` | Object | **标准指标**，壳子看板内置图表展示 |
| `base.request_count` | Number | 累计请求量 |
| `base.error_rate` | Number | 错误率 (0~1) |
| `base.avg_response_ms` | Number | 平均响应时间(毫秒) |
| `base.cpu_percent` | Number | CPU 使用率 (%) |
| `base.memory_mb` | Number | 内存占用 (MB) |
| `custom` | Object | **自定义指标**，可以随意添加 key-value |

### 5.3 代码示例（Python / FastAPI）

```python
import time
import psutil  # pip install psutil

# 请求统计（通过中间件采集）
_request_count = 0
_error_count = 0
_total_response_ms = 0.0
_started_at = time.time()


@app.middleware("http")
async def _metrics_middleware(request: Request, call_next):
    global _request_count, _error_count, _total_response_ms
    _request_count += 1
    start = time.time()
    response = await call_next(request)
    elapsed_ms = (time.time() - start) * 1000
    _total_response_ms += elapsed_ms
    if response.status_code >= 400:
        _error_count += 1
    return response


@app.get("/metrics")
def metrics():
    global _request_count, _error_count, _total_response_ms
    avg_ms = round(_total_response_ms / _request_count, 2) if _request_count > 0 else 0
    error_rate = round(_error_count / _request_count, 3) if _request_count > 0 else 0.0

    # CPU / 内存指标（依赖 psutil，不可用时返回 -1）
    try:
        proc = psutil.Process()
        cpu = round(proc.cpu_percent(interval=0.1), 1)
        mem = round(proc.memory_info().rss / (1024 * 1024), 1)
    except Exception:
        cpu, mem = -1.0, -1.0

    return {
        "status": "UP",
        "timestamp": int(time.time()),
        "base": {
            "request_count": _request_count,
            "error_rate": error_rate,
            "avg_response_ms": avg_ms,
            "cpu_percent": cpu,
            "memory_mb": mem,
        },
        "custom": {
            # 你可以在此添加业务自定义指标
            # "file_processed_today": 123,
        },
    }
```

> **注意**：
> - `cpu_percent` / `memory_mb` 需要 `psutil` 库，如未安装应返回 `-1` 而非崩溃
> - 通过 FastAPI 中间件 (`@app.middleware("http")`) 统计请求量，无需在每个接口埋点

### 5.4 自定义指标说明

`custom` 中的字段完全由你决定，可以随时新增。例如：

```json
"custom": {
  "file_processed_today": 1024,
  "avg_parse_time_ms": 350,
  "queue_size": 42
}
```

- 数值类型指标会被自动识别，可在壳子看板中生成趋势图
- 字符串类型指标以表格展示最新值
- **你可以随时新增字段，不需要等壳子发版**

## 六、步骤 4：前端支持 iframe 嵌入

### 6.1 壳子如何加载你的前端

```
用户点击菜单 "你的模块" → 壳子路由 /app/your-app-key/home
                              ↓
壳子生成 iframe:
  <iframe src="http://你的前端地址/home?token={JWT_TOKEN}" />
                              ↓
你的前端接收 token → 解析 → 建立登录态 → 正常渲染
```

### 6.2 你的前端需要做什么

**1. 从 URL 解析 token**

```javascript
// 在应用入口处解析
function getToken() {
  const params = new URLSearchParams(window.location.search);
  return params.get('token');
}
```

**2. 监听 postMessage 获取上下文**

壳子会在 iframe 加载完成后通过 `postMessage` 下发上下文信息：

```javascript
// main.jsx / App.vue / app.tsx 等入口文件中
window.addEventListener('message', (event) => {
  if (event.data?.type === 'SHELL_CONTEXT') {
    const { token, theme, locale, userId } = event.data.payload;
    console.log('[壳子上下文]', { token, theme, locale, userId });

    // 保存 token 用于后续 API 请求
    localStorage.setItem('token', token);

    // 可选：适配壳子主题
    if (theme === 'dark') {
      document.documentElement.setAttribute('data-theme', 'dark');
    }
  }
});
```

**3. 携带 token 发起 API 请求**

```javascript
// 你的所有后端 API 请求都带上 token
fetch('http://你的后端/your-api', {
  headers: {
    'Authorization': `Bearer ${getToken()}`,
    'Content-Type': 'application/json'
  }
});
```

### 6.3 完整示例（React）

```jsx
// main.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

function getToken() {
  return new URLSearchParams(window.location.search).get('token');
}

// 监听壳子上下文
window.addEventListener('message', (event) => {
  if (event.data?.type === 'SHELL_CONTEXT') {
    localStorage.setItem('token', event.data.payload.token);
  }
});

// 初始化时从 URL 获取 token
const token = getToken();
if (token) {
  localStorage.setItem('token', token);
}

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

### 6.4 Vue 示例

```javascript
// main.js
import { createApp } from 'vue';
import App from './App.vue';

const app = createApp(App);

// 从 URL 获取 token
const params = new URLSearchParams(window.location.search);

  }
});

app.mount('#app');
```

### 6.5 前端开发时的注意事项

1. **不要阻止 iframe 嵌入**：确保你的前端没有设置 `X-Frame-Options: DENY` 或 `Content-Security-Policy: frame-ancestors 'none'`
2. **路由使用 Hash 或 Browser 路由均可**：壳子通过 `?token=xxx` 传参，与你自己的路由不冲突
3. **本地开发联调**：开发时可直接用浏览器打开你的前端页面，手动加 `?token=xxx` 模拟壳子环境

### 6.6 Vanilla JS / 静态 HTML 页面专用接入方案

如果你的子应用是**原生 HTML + CSS + JavaScript**（无 React/Vue 等框架），我们提供了开箱即用的 `hras-init.js` 适配脚本，只需在页面底部引入一行即可完成所有接入工作。

#### 6.6.1 一行引入

```html
<!-- 在 </body> 前引入 -->
<script src="hras-init.js"></script>
```

完整示例（`your-page.html`）：

```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <title>你的模块</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <header class="app-header">
      <!-- 你的页面 header -->
    </header>
    <main>...</main>

    <script src="your-biz.js"></script>
    <script src="hras-init.js"></script>  <!-- 最后引入 -->
  </body>
</html>
```

#### 6.6.2 hras-init.js 自动完成的工作

引入后无需任何额外代码，脚本会自动处理：

| 功能 | 说明 |
|------|------|
| Token 解析 | 从 URL `?token=xxx` 自动提取并存储到 `localStorage` |
| 壳子上下文 | 监听 `postMessage` 接收 token / theme / locale / userId |
| 嵌入检测 | `window.self !== window.top` 时自动标记 `body.embedded` |
| fetch 拦截 | 自动为同源 API 请求注入 `Authorization: Bearer {token}` 头 |

子应用业务代码无需改动：

```javascript
// 你的业务代码照常使用 fetch，token 会自动注入
fetch('/api/your-endpoint', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ ... })
});

// 也可以手动读取
const token = window.__hrasToken;
```

#### 6.6.3 嵌入模式 CSS 适配

页面在 iframe 中加载时，`<body>` 会自动添加 `embedded` 类。你可以在 CSS 中按需适配布局，但**不建议隐藏 header**，否则会导致布局错乱：

```css
/* ✅ 推荐：嵌入时保持 header 可见，只调整内边距 */
body.embedded {
  padding-top: 0;
}

body.embedded .labor-stage {
  padding-top: 20px;  /* 合理压缩而不是隐藏 */
}

/* ❌ 不推荐：隐藏 header 会导致 grid 布局缺列 */
/* body.embedded .app-header { display: none; } */
```

> **踩坑经验**：如果页面使用了 CSS Grid 布局（如 `grid-template-columns: 260px 1fr auto`），隐藏某一列会破坏整体布局。嵌入模式下让所有元素保持可见，仅通过 padding 等非破坏性方式压缩间距。

#### 6.6.4 后端移除 X-Frame-Options（FastAPI 示例）

静态 HTML 页面通常由后端直接服务，需要确保响应头允许 iframe 嵌入：

```python
from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")
async def _allow_iframe_middleware(request: Request, call_next):
    response = await call_next(request)
    # 移除 X-Frame-Options，允许被壳子 iframe 嵌入
    response.headers.pop("X-Frame-Options", None)
    response.headers.pop("x-frame-options", None)
    return response
```

> **注意**：iframe 模式**不需要 CORS 配置**，子应用 CSS/JS 由同源加载。之前尝试过的 wujie fiber 模式需要 CORS，但对 Vanilla JS 页面有兼容性问题（全局变量重复声明、DOM 未就绪时查询等），因此最终选择原生 iframe。

#### 6.6.5 微前端方案选型参考

对于 Vanilla JS 子应用，不同微前端方案的适用性对比：

| 方案 | Vanilla JS 适用性 | 改造量 | 隔离性 |
|------|:--:|------|:--:|
| **iframe（当前方案）** | ✅ 完全兼容 | 零改造 | 完全隔离 |
| wujie fiber | ❌ DOM 查询在时机问题 | 需生命周期改造 | Proxy 沙箱 |
| qiankun | ❌ 需导出 bootstrap/mount/unmount | 大量改造 | Proxy 沙箱 |
| Module Federation | ❌ 必须 Webpack 5 构建 | 完全重写为 React | 无隔离 |

> 如果你的子应用是 React/Vue SPA，可以评估 wujie 或 qiankun。如果是 Vanilla JS，**iframe 是最务实的选择**。

## 七、步骤 5（可选）：验证壳子 JWT Token

### 7.1 什么时候需要验证

如果你的后端接口需要知道当前登录用户是谁，就需要解析壳子签发的 JWT Token。

### 7.2 JWT Token 结构

```
Header: { "alg": "HS512" }
Payload: { "userId": 1, "username": "admin", "sub": "admin", "iat": ..., "exp": ... }
```

### 7.3 验证方式

**方案一：对称密钥验证（推荐）**

联系壳子平台管理员获取 JWT 签名密钥（`hras-shell-jwt-secret-key-must-be-at-least-256-bits-long-for-hs256`），在你的后端做本地验证。

```java
// Spring Boot 中验证 JWT
import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.security.Keys;

String secret = "hras-shell-jwt-secret-key-must-be-at-least-256-bits-long-for-hs256";
var key = Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));

var claims = Jwts.parser()
    .verifyWith(key)
    .build()
    .parseSignedClaims(token)
    .getPayload();

Long userId = claims.get("userId", Long.class);
String username = claims.getSubject();
```

**方案二：调用壳子接口验证**

对于不方便保管密钥的语言，可以调用壳子 `/api/auth/me` 接口验证：

```
GET /api/auth/me
Authorization: Bearer {token}
```

壳子返回当前用户信息，验证通过。

### 7.4 模块内权限管理

你的模块可以：
- **复用壳子用户体系**：根据 token 中的 `userId` 做自己的权限控制
- **建立映射关系**：将壳子 `userId` 映射到你自己的用户表
- **完全独立**：不使用壳子用户，自己管理登录和权限

## 七-A、步骤 6（可选）：操作日志上报

> 版本：v1.0 | 更新时间：2026-06-13

### 概述

壳子平台提供**操作日志采集**功能，可以统一查看所有子应用的操作行为。你只需做**零到极少的改造**即可接入。

接入后，壳子后台的"操作日志"页面将自动展示你模块的请求量、成功/失败分布、每个 API 的调用详情。

### 方案对比

| 方案 | 改造量 | 覆盖范围 | 适用场景 |
|------|:--:|------|----------|
| **A：SDK 前端埋点**（推荐） | 一行代码 | 前端所有 fetch/XHR 请求 | 快速接入，立即见效 |
| **B：后端拦截器** | ~30 行配置+代码 | 后端所有 Controller 请求（含定时任务） | 需要完整服务端日志 |

**推荐两个方案同时使用**，SDK 覆盖前端操作，拦截器补充后端操作。

### 方案 A：SDK 前端埋点（一行代码）

如果你已经按第十二章接入了 `hras-sdk.js`，只需在初始化时添加一行：

```javascript
// ✅ 设置当前模块标识，SDK 自动采集所有 fetch/XHR 请求并上报
__HRAS__.setModuleKey('your-app-key');
```

完整示例（React）：

```jsx
// src/main.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

async function bootstrap() {
  await window.__HRAS__.ready();
  __HRAS__.setModuleKey('your-app-key');  // ← 加这一行
  const user = window.__HRAS__.getUser();

  ReactDOM.createRoot(document.getElementById('root')).render(
    <React.StrictMode>
      <App user={user} />
    </React.StrictMode>
  );
}
bootstrap();
```

SDK 会自动完成：

| 能力 | 说明 |
|------|------|
| 采集时机 | 每个 fetch/XHR 请求完成时 |
| 采集内容 | 请求方法、路径、状态码、耗时 |
| 批量上报 | 累积 20 条或 5 秒自动发送 |
| 上报通道 | iframe 模式 → postMessage；独立模式 → POST /api/audit/report |
| 敏感过滤 | 自动忽略静态资源（.js/.css/.png 等） |

**不需要**采集请求体或响应体，保证数据安全。

### 方案 B：后端拦截器（Spring Boot 示例）

如果你的子应用是 Spring Boot 项目，只需复制两个文件即可接入。

#### 环境变量配置

在 `application.yml` 中添加：

```yaml
hras:
  shell:
    url: ${HRAS_SHELL_URL:http://localhost:8066}
  module:
    key: ${HRAS_MODULE_KEY:your-app-key}
    api-key: ${HRAS_MODULE_API_KEY:}   # 注册后壳子返回的 apiKey
```

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `hras.shell.url` | 壳子后端地址 | `http://localhost:8066` |
| `hras.module.key` | 你的模块标识 | `your-app-key` |
| `hras.module.api-key` | 注册后壳子返回的 apiKey | 空（不填则不上报） |

> `api-key` 由壳子在模块注册时自动生成并返回。你可以在注册成功回调中读取并设置到环境变量。

#### 添加拦截器类

在你的项目中新建 `AuditReportInterceptor.java`：

```java
package com.yourmodule.config;

import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;
import org.springframework.web.servlet.HandlerInterceptor;

import java.io.OutputStream;
import java.net.HttpURLConnection;
import java.net.URI;
import java.nio.charset.StandardCharsets;
import java.time.Instant;

@Component
public class AuditReportInterceptor implements HandlerInterceptor {

    private static final Logger log = LoggerFactory.getLogger(AuditReportInterceptor.class);

    @Value("${hras.shell.url:http://localhost:8066}")
    private String shellUrl;

    @Value("${hras.module.key:}")
    private String moduleKey;

    @Value("${hras.module.api-key:}")
    private String apiKey;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) {
        request.setAttribute("audit_start_time", System.currentTimeMillis());
        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) {
        String path = request.getRequestURI();
        if (path.startsWith("/health") || path.startsWith("/metrics") || path.startsWith("/actuator")) {
            return;
        }
        if (moduleKey.isEmpty()) return;

        int status = response.getStatus();
        long duration = 0L;
        Object startObj = request.getAttribute("audit_start_time");
        if (startObj instanceof Long) {
            duration = System.currentTimeMillis() - (Long) startObj;
        }

        String result = status >= 400 ? "FAILURE" : "SUCCESS";
        String operator = request.getHeader("X-Employee-No");
        if (operator == null || operator.isEmpty()) {
            operator = request.getRemoteUser();
        }
        if (operator == null || operator.isEmpty()) {
            operator = "system";
        }

        String json = String.format(
            "{\"logs\":[{" +
            "\"moduleKey\":\"%s\",\"operator\":\"%s\",\"operation\":\"%s\"," +
            "\"target\":\"%s\",\"requestPath\":\"%s\",\"requestMethod\":\"%s\"," +
            "\"result\":\"%s\",\"duration\":%d,\"timestamp\":\"%s\"}]}",
            esc(moduleKey), esc(operator), esc(status >= 400 ? "请求失败" : "请求成功"),
            esc(path), esc(path), esc(request.getMethod()),
            esc(result), duration, Instant.now().toString()
        );

        new Thread(() -> {
            try {
                URI uri = URI.create(shellUrl + "/api/audit/report");
                HttpURLConnection conn = (HttpURLConnection) uri.toURL().openConnection();
                conn.setRequestMethod("POST");
                conn.setDoOutput(true);
                conn.setConnectTimeout(3000);
                conn.setReadTimeout(3000);
                conn.setRequestProperty("Content-Type", "application/json;charset=UTF-8");
                if (apiKey != null && !apiKey.isEmpty()) {
                    conn.setRequestProperty("Authorization", "Bearer " + apiKey);
                }
                try (OutputStream os = conn.getOutputStream()) {
                    os.write(json.getBytes(StandardCharsets.UTF_8));
                }
            } catch (Exception e) {
                log.debug("操作日志上报异常: {}", e.getMessage());
            }
        }).start();
    }

    private static String esc(String s) {
        if (s == null) return "";
        return s.replace("\\", "\\\\").replace("\"", "\\\"");
    }
}
```

#### 注册拦截器

在 `WebConfig.java` 中注册：

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {

    private final AuditReportInterceptor auditReportInterceptor;

    public WebConfig(AuditReportInterceptor auditReportInterceptor) {
        this.auditReportInterceptor = auditReportInterceptor;
    }

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(auditReportInterceptor)
                .addPathPatterns("/api/**")
                .excludePathPatterns("/health", "/metrics", "/actuator/**");
    }
}
```

#### 非 Spring Boot 项目？

| 语言/框架 | 实现方式 |
|------|------|
| Go / Gin | 中间件（middleware），请求后异步 POST |
| Python / FastAPI | `@app.middleware("http")`，用 `httpx` 异步上报 |

> 核心逻辑一致：记录请求开始时间 → Controller 执行 → 收集 method/path/status/duration → 异步 POST 到 `{壳子地址}/api/audit/report`，携带 `Authorization: Bearer {apiKey}`。

## 九、常见问题

### Q1：注册失败返回 401？

返回 401 表示 apiKey 认证失败，请检查：
- `Authorization: Bearer {apiKey}` 头是否正确设置
- apiKey 是否从管理员处正确获取，注意不要有多余空格
- 模块标识是否已被作废

### Q1-2：注册失败返回 400？

检查 `Content-Type` 是否为 `application/json; charset=utf-8`，以及请求体 JSON 格式是否正确。如果你使用 `httpx` 的 `json=` 参数，它会自动设置正确的 Content-Type。

### Q1-1：启动时完全没有注册日志输出？

这是最常见的踩坑问题，按以下顺序排查：

1. **检查注册开关**：确认 `HRAS_SHELL_REGISTER_ENABLED` 不是 `false`（该环境变量默认为 `true`）
2. **检查代码是否被引入**：确认 `@app.on_event("startup")` 装饰的函数确实在应用的入口模块中
3. **确认日志输出方式**：如果使用 `logging`，确保 level 足够低（`logging.INFO`）；推荐使用 `print()` 直接输出到终端

### Q2：注册成功但壳子看不到模块？
检查模块状态是否为"启用"。在壳子后台 "模块管理" 页面查看。

### Q3：iframe 加载空白？
- 检查后端是否设置了 `X-Frame-Options: DENY`，需要通过中间件移除（FastAPI 示例见 6.6.4）
- 检查前端 URL 是否可被壳子服务网络访问
- 打开浏览器开发者工具查看 iframe 请求是否被拦截
- 如果是 Vanilla JS 页面，确认 `hras-init.js` 已正确引入且路径正确

### Q4：中文名称乱码？
确保请求头 `Content-Type` 包含 `charset=utf-8`，且请求体编码为 UTF-8。

### Q5：模块之间如何共享数据？
模块之间**不直接通信**。如需共享数据，请通过你自己的后端 API 或数据库实现。

### Q6：我的前端端口和壳子不同，需要配置跨域吗？
不需要。你的前端通过 iframe 加载，API 请求直接发到你的后端，不经过壳子代理。

### Q7：启动时报 `PermissionError: [Errno 1] Operation not permitted`？

这不是注册问题，而是你的终端当前工作目录被删除或不可访问。解决方法：先 `cd` 到项目根目录再启动：

```bash
cd /path/to/your-project
python3 -m uvicorn your_module.app:app --reload --port 8066
```

### Q8：FastAPI 使用 `requests` 库卡住不动？

`requests` 是**同步**库，在 FastAPI 的异步 `on_event("startup")` 中调用会阻塞事件循环。必须改用异步库 **`httpx`**：

```python
# ❌ 错误：requests 是同步的，会阻塞
import requests
requests.post(...)

# ✅ 正确：httpx 是异步的
import httpx
async with httpx.AsyncClient() as client:
    await client.post(...)
```

### Q9：iframe 嵌入后页面 header 样式丢失/布局错乱？

这是一个高频踩坑点，通常由两个原因导致：

**原因一**：`hras-init.js` 检测到嵌入后给 `<body>` 添加 `embedded` 类，CSS 中定义了 `body.embedded .app-header { display: none; }` 导致 header 被隐藏。

**解决**：删除这类 `display: none` 规则。嵌入模式下保持 header 可见，仅用 padding 压缩间距。

**原因二**：页面使用了 CSS Grid 布局（如 `.labor-audit-header { grid-template-columns: 260px 1fr auto }`），header 中 logo 被 CSS 隐藏后，第一列变成空白，剩余两列位置偏移。

**解决**：嵌入时不隐藏 header 内的任何子元素。如果需要调整布局，使用完整的 `grid-template-columns` 重新定义列宽，而非隐藏某一列。

> **核心原则**：嵌入模式只做**压缩**（padding/margin 减小），不做**删减**（display: none）。

## 十、独立部署与壳子模式切换

你的模块应同时支持**两种运行模式**，无需修改代码，只通过环境变量切换：

| 模式 | `HRAS_SHELL_REGISTER_ENABLED` | 行为 |
|---|---|---|
| **壳子模式** | `true`（默认） | 启动时向壳子自动注册；前端通过 iframe 嵌入 |
| **独立模式** | `false` | 跳过注册，独立运行（用于本地开发/测试/演示） |

### 前端适配

通过引入 `hras-init.js` 即可自动适配（详见第六章 6.6 节），无需手写检测逻辑。脚本会自动：

- 检测 `window.self !== window.top` 判断嵌入模式
- 从 URL 解析 token 并存储到 `localStorage`
- 监听 `postMessage` 接收壳子上下文
- 拦截 `fetch` 自动注入 `Authorization` 头

**嵌入模式 CSS 适配要点**：

```css
/* ✅ 推荐：嵌入时压缩间距，保持所有元素可见 */
body.embedded .labor-stage {
  padding-top: 20px;
}

/* ❌ 不推荐：隐藏 header 导致 Grid 布局缺列，页面布局错乱 */
/* body.embedded .app-header { display: none; } */
/* body.embedded .portal-logo-mark { display: none; } */
```

> **原则**：嵌入模式下通过 padding/margin 等非破坏性方式压缩间距，**不要使用 `display: none`** 隐藏 header 中的关键元素，否则可能导致 `grid-template-columns` 布局缺列错位。

### 切换示例

```bash
# 壳子模式
export HRAS_SHELL_REGISTER_ENABLED=true  # 默认值，可不设
python3 -m uvicorn app:app --port 8000

# 独立模式
export HRAS_SHELL_REGISTER_ENABLED=false
python3 -m uvicorn app:app --port 8000
```

> **核心原则**：模块感知不到壳子的存在也能正常工作，壳子注册只是一个可选的"上报"动作。

## 十二、SDK 接入：一键获取登录用户信息

> 版本：v2.0 | 更新时间：2026-06-11

### 12.1 为什么要接入 SDK

之前第六章介绍的前端接入方式需要每个子应用**手写** token 解析、postMessage 监听、API 拦截器。接入 SDK 后这些工作**零代码完成**：

| 能力 | 手写接入（第六章） | SDK 接入（本章） |
|------|:--:|:--:|
| 从 URL 解析 token | 需手写 | ✅ 自动 |
| 监听 SHELL_CONTEXT 获取用户信息 | 需手写 | ✅ 自动 |
| fetch / XHR 自动注入 token | 需手写 | ✅ 自动 |
| 获取 userId / username / realName | ❌ 不支持 | ✅ `__HRAS__.getUser()` |
| 壳子 ↔ 子应用双向通信 | ❌ 不支持 | ✅ `__HRAS__.requestContext()` |
| iframe 嵌入检测（body.embedded） | ✅ 已有 | ✅ 兼容 |
| 重复执行不产生副作用（幂等） | ❌ | ✅ |

### 12.2 接入方式

#### 方式 A（推荐）：`<script>` 标签引入

在你的 HTML 入口文件（如 `index.html`）中，**在所有业务 JS 之前**引入：

```html
<script src="https://<hras-host>/sdk/hras-sdk.js"></script>
```

本地开发时可使用壳子本地地址：

```html
<script src="/sdk/hras-sdk.js"></script>
```

#### 方式 B：复制到项目中（离线/内网）

将 `hras-sdk.js` 文件复制到你的项目静态资源目录，然后：

```html
<script src="/hras-sdk.js"></script>
```

### 12.3 AI 自动接入指令

你可以直接将以下指令发送给 AI（如 Claude Code / Cursor / Copilot），让 AI 自动完成接入：

> **请帮我在当前子应用项目中接入 hras-sdk。按照以下步骤操作，每步先检测是否已完成，已完成的跳过：**
>
> 1. **检测 SDK 是否已引入**：在 HTML 入口文件中搜索 `hras-sdk.js`。如果已存在 `<script src="...hras-sdk.js">`，跳过步骤 2。
> 2. **引入 SDK 脚本**：在 HTML 入口文件的 `<head>` 或 `<body>` 底部添加 `<script src="/hras-sdk.js"></script>`，确保在所有业务 JS 之前加载。
> 3. **检测旧版 hras-init.js**：如果项目中已有 `hras-init.js`，将其替换为 `hras-sdk.js`（新版向下兼容，`window.__hrasToken` 等全局变量保持不变）。
> 4. **检测手写 token 解析**：搜索代码中是否有 `URLSearchParams('token')` 或 `getToken()` 等手写的 token 解析逻辑。如果有且只是读取 token + 存 localStorage，可以删除，SDK 已自动处理。
> 5. **检测 fetch/XHR 拦截器**：搜索代码中是否有手写的 `Authorization: Bearer` 注入拦截器。如果有且只做注入，可以删除。
> 6. **检测 postMessage 监听**：搜索 `SHELL_CONTEXT` 字符串。如果已有手写的 `addEventListener('message', ...)` 监听此事件，将其中的 `token` 读取替换为 `__HRAS__.getToken()`，`user` 读取替换为 `__HRAS__.getUser()`。
> 7. **验证**：在控制台执行 `__HRAS__.getUser()`，应返回包含 `userId`、`username`、`realName` 等字段的对象。

> **重要规则**：
> - 每一步操作前先检测是否已存在，已存在则跳过
> - 不要破坏已有的业务逻辑
> - 如果没有 HTML 入口文件（纯 API 后端项目），跳过前端接入步骤

### 12.4 幂等检查清单（AI 执行参考）

以下是 AI 自动接入时的逐项检测清单。**每项先检查 → 已满足则跳过 → 未满足才执行**：

| 检查项 | 检测方式 | 已满足的条件 | 未满足时的操作 |
|--------|----------|-------------|---------------|
| ① SDK 脚本已引入 | 搜索 HTML 中 `hras-sdk.js` | 存在任一引用 | 添加 `<script>` 标签 |
| ② 无旧版 hras-init.js | 搜索 `hras-init.js` | 不存在 | 存在则替换为 `hras-sdk.js` |
| ③ token 从 URL 解析 | 搜索 `URLSearchParams` + `token` | 无手写逻辑 或 仅有 SDK | 删除手写逻辑 |
| ④ fetch/XHR 拦截器 | 搜索 `Authorization.*Bearer` 拦截 | 无手写逻辑 | 删除手写逻辑 |
| ⑤ SHELL_CONTEXT 监听 | 搜索 `SHELL_CONTEXT` | 不存在或已用 SDK API | 替换为 `__HRAS__.getUser()` |
| ⑥ embedded 检测 | 搜索 `body.embedded` | 存在类名使用 | SDK 已自动添加 |

> **核心原则**：SDK 是已有手写接入代码的**替代品**，不是补充品。已接入的代码能删则删，保持代码库干净。

### 12.5 各框架接入示例

#### 12.5.1 Vanilla HTML / 静态页面

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="utf-8" />
  <title>我的子应用</title>
  <!-- ✅ SDK 在 head 中引入，最早加载 -->
  <script src="/hras-sdk.js"></script>
  <link rel="stylesheet" href="styles.css" />
</head>
<body>
  <div id="app"></div>

  <!-- 业务 JS 在 SDK 之后 -->
  <script>
    // ✅ SDK 已就绪，直接使用
    async function main() {
      await __HRAS__.ready();
      const user = __HRAS__.getUser();
      console.log('当前用户:', user.realName, user.userId);

      // 发起 API 请求，token 自动注入
      const res = await fetch('/api/data');
      const data = await res.json();
      // ...
    }
    main();
  </script>
</body>
</html>
```

#### 12.5.2 React（Vite / CRA）

**1. 在 `index.html` 中引入 SDK：**

```html
<!-- index.html -->
<!doctype html>
<html lang="zh-CN">
<head>
  <script src="/hras-sdk.js"></script>
</head>
<body>
  <div id="root"></div>
  <script type="module" src="/src/main.jsx"></script>
</body>
</html>
```

**2. 在入口 `main.jsx` 中等待就绪后渲染：**

```jsx
// src/main.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

async function bootstrap() {
  // 等待壳子上下文就绪
  await window.__HRAS__.ready();

  const user = window.__HRAS__.getUser();
  const token = window.__HRAS__.getToken();

  ReactDOM.createRoot(document.getElementById('root')).render(
    <React.StrictMode>
      <App user={user} token={token} />
    </React.StrictMode>
  );
}

bootstrap();
```

**3. 配置 axios（可选）：**

```js
// src/api/client.js
import axios from 'axios';

const client = axios.create({ baseURL: '/api' });

client.interceptors.request.use((config) => {
  const token = window.__HRAS__.getToken();
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

export default client;
```

#### 12.5.3 Vue（Vite / CLI）

**1. 在 `index.html` 中引入 SDK（同 React）。**

**2. 在 `main.js` 中等待就绪后挂载：**

```js
// src/main.js
import { createApp } from 'vue';
import App from './App.vue';

async function bootstrap() {
  await window.__HRAS__.ready();
  const user = window.__HRAS__.getUser();

  const app = createApp(App);
  app.provide('hrasUser', user);  // 注入全局用户信息
  app.mount('#app');
}

bootstrap();
```

#### 12.5.4 从 hras-init.js 迁移

如果项目中已经使用了第六章介绍的 `hras-init.js`，迁移只需两步：

**步骤 1**：替换 `<script>` 标签

```diff
- <script src="hras-init.js"></script>
+ <script src="/hras-sdk.js"></script>
```

**步骤 2**：确认向后兼容的全局变量仍然可用

```js
// 以下 API 在新旧 SDK 中行为一致，无需修改业务代码
window.__hrasToken  // ✅ 仍可用
document.body.classList.contains('embedded')  // ✅ 仍可用
```

### 12.6 验证方法

在壳子 iframe 中打开你的子应用页面，打开浏览器控制台执行：

```js
// 1. 检查 token
console.log('token:', __HRAS__.getToken());
// 应输出 jwt token 字符串

// 2. 检查用户信息
console.log('user:', __HRAS__.getUser());
// 应输出 { userId: 1, username: "zhangsan", realName: "张三", roleName: "管理员", modulePermissions: "..." }

// 3. 检查嵌入检测

// 在壳子 iframe 中应输出 true

// 4. 检查 fetch 拦截
fetch('/api/some-endpoint');
// 在 Network 面板中查看请求头，应自动带 Authorization: Bearer <token>
```

### 12.7 SDK 完整 API 参考

```js
// ── 全局入口 ──────────────────────────
window.__HRAS__
// 所有 API 都通过 __HRAS__ 访问

// ── 就绪等待 ──────────────────────────
await __HRAS__.ready()
// Promise，壳子上下文就绪后 resolve。
// 如果 1.5 秒内未收到 SHELL_CONTEXT，SDK 会自动向壳子发送 SHELL_CONTEXT_REQUEST。

// ── 数据获取 ──────────────────────────
__HRAS__.getUser()
// → { userId, username, realName, roleName, modulePermissions } | null

__HRAS__.getToken()
// → "jwt-string" | null

// ── 主动通信 ──────────────────────────
__HRAS__.requestContext()
// 向壳子发送 SHELL_CONTEXT_REQUEST，壳子收到后重新下发 SHELL_CONTEXT。

// ── 安全配置 ──────────────────────────
__HRAS__.setOriginWhitelist(['https://hras.example.com'])
// 设置 postMessage origin 白名单，仅接受来自指定源的 SHELL_CONTEXT。

// ── 销毁（测试用） ────────────────────
__HRAS__.destroy()
// 移除所有事件监听和拦截器，恢复 window.fetch 和 window.XMLHttpRequest 原始实现。

// ── 向后兼容（与 hras-init.js 一致） ──
window.__hrasToken   // token 字符串
window.__hrasUser    // 用户对象
```

### 12.8 SDK 通信时序

```
子应用加载
    │
    ├── SDK 立即提取 URL 中的 ?token=xxx
    ├── SDK 注册 postMessage 监听 ─────────────┐
    ├── SDK 包装 fetch / XMLHttpRequest         │
    │                                          │
    ▼ (1.5s 后)                                │
SDK 发送 SHELL_CONTEXT_REQUEST ──────────┐      │
    │                                    │      │
    │                      壳子收到请求   │      │
    │                      壳子调用 sendContext()
    │                      下发 SHELL_CONTEXT  │
    │                          │               │
    │                          ▼               │
    └──────────────────── SDK 收到 ◄───────────┘
                                │
                                ▼
                        _user / _token 填充
                        _readyResolve() 触发
                        __HRAS__.ready() resolve
```
