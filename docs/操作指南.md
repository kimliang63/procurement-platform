# 采购协同平台 V2 操作指南

## 一、系统访问

| 环境 | 地址 |
|------|------|
| Web 端 | https://procurement-platform-rosy.vercel.app |
| 后端 API | https://procurement-server-production-b325.up.railway.app |

首次使用点击「飞书登录」，通过飞书 OAuth 授权后自动进入系统。第一个登录的用户自动成为管理员。

## 二、Web 端操作

### 2.1 项目看板（首页）

进入系统默认显示项目看板，包含：

- **统计卡片**：本年累计项目数、进行中项目、已定标项目、100万元以上项目
- **各BU执行中项目**：柱状图展示 FBU/LBU/ABU 各部门进行中的项目数
- **各采购员负责的项目数量**：横向柱状图
- **年度采购金额**：饼图展示各部门金额占比
- **任务类型**：饼图展示框架招标/单一来源/单次采购的分布

### 2.2 创建项目

1. 进入「项目列表」页面，点击右上角「创建项目」
2. 填写以下信息：
   - **项目名称**（必填）：如"XX设备采购"
   - **采购品类**（必填）：设备/材料/服务/其他
   - **所属部门**（必填）：FBU/LBU/ABU
   - **负责人**（必填）：从用户列表选择
   - **采购金额**（必填）：单位万元
   - **任务类型**（必填）：
     - 框架招标 → 招标流程（12个节点必填）
     - 单一来源 → 快速流程（4个节点必填）
     - 单次采购＜100万 → 快速流程
     - 单次采购≥100万 → 招标流程
   - **计划开始/结束**（必填）
   - **备注**（选填）
3. 点击确定，系统自动编号（CG-2026-XXX）并初始化 15 个阶段节点

### 2.3 项目列表（时间线视图）

- 每个项目卡片展示 15 个阶段的时间线进度条
- 颜色含义：🟢已完成 🔵进行中 ⚪待开始 🔴异常/逾期
- 点击项目卡片进入详情页
- 支持按名称搜索、按负责人筛选

### 2.4 项目详情

**基础信息区**：负责人、采购金额、任务类型、所属部门、品类、计划周期

**节点进度 Tab**：
- 顶部时间线进度条，点击节点可编辑
- 节点表格：阶段 | 状态 | 计划开始 | 计划结束 | 实际完成 | 问题数 | 操作
- 点击「编辑」可修改节点的计划日期、实际完成日期、备注
- 点击「创建问题」可为该节点添加问题

**问题列表 Tab**：
- 展示该项目下的所有问题
- 可直接修改问题状态（待处理/处理中/已关闭）

### 2.5 问题追踪

独立的问题管理页面，支持：
- 按状态筛选（待处理/处理中/已关闭）
- 按优先级筛选（高/中/低）
- 创建问题时关联项目和阶段
- 编辑和删除问题

### 2.6 用户管理（仅管理员）

- 查看所有已注册用户
- 修改用户角色：管理员/项目经理/成员
- 第一个通过飞书登录的用户自动成为管理员

## 三、飞书 Bot 操作

### 3.1 添加 Bot

在飞书应用管理后台将「采购项目小助手」添加到工作群。

### 3.2 群聊绑定/解绑项目

**绑定项目**：在群聊中发送：
```
@采购项目小助手 绑定 XX设备采购项目
```
一个群只能绑定一个项目。绑定后群内的 Bot 操作都针对该项目。

**解绑项目**：在群聊中发送：
```
@采购项目小助手 解绑
```
解除当前群与项目的绑定关系。解绑后可重新绑定其他项目。

### 3.3 自然语言指令

Bot 支持自然语言交互，常用指令：

| 操作 | 示例对话 |
|------|---------|
| 创建项目 | "创建一个新项目，名称是CC服务采购，品类是服务，部门FBU，预算100万，明年全年" |
| 查看项目 | "有哪些项目" / "查看XX项目" |
| 查看节点 | "XX项目到哪一步了" / "查看项目节点" |
| 推进节点 | "定标完成了" / "需求确认这个节点完成了" |
| 标记异常 | "定标有问题" / "技术交流出问题了" |
| 更新信息 | "XX项目的预算改成200万" |
| 创建问题 | "XX设备采购的供应商C报价超预算了" |

### 3.4 创建项目流程

1. 发送创建指令（如"创建一个新项目"）
2. Bot 追问缺少的信息（名称、品类、部门、预算、日期）
3. 信息完整后 Bot 发送确认卡片（显示项目详情）
4. 点击卡片「确认创建」按钮完成创建（点击前会自动检查项目名是否重复）
5. 群聊中创建的项目会自动绑定到当前群

### 3.5 节点操作

Bot 识别到节点操作意图后：
- 自动关联当前绑定的项目（群聊）或最后查询的项目（私聊）
- 标记完成后显示下一阶段提示
- 无计划日期时提示需要先设置日期

### 3.6 权限说明

| 场景 | 负责人 | 非负责人 |
|------|--------|---------|
| 群聊查询 | ✅ | ✅ |
| 群聊创建/更新/推进节点 | ✅ | ❌ 提示"仅负责人可操作" |
| 私聊操作自己负责的项目 | ✅ | — |
| 私聊操作他人项目 | ❌ | — |

### 3.7 周报

发送"发周报"获取当前绑定项目的本周变化，包括：
- 项目基本信息和进度
- 本周有日期变动的节点（实际完成、计划开始/结束）

管理员发送"管理周报"可查看全局周报。

## 四、节点阶段说明（15个）

| 序号 | 阶段 | 说明 |
|------|------|------|
| 1 | 需求确认 | 项目启动，确认采购需求 |
| 2 | 供应商开发 | 寻找和评估潜在供应商 |
| 3 | 技术交流 | 与供应商进行技术方案沟通 |
| 4 | 打样 | 供应商提供样品测试 |
| 5 | 招标方案审批 | 招标方案内部审批 |
| 6 | 发标 | 发布招标文件 |
| 7 | 答疑 | 回答供应商疑问 |
| 8 | 供应商回标 | 供应商提交投标文件 |
| 9 | 开标 | 开启投标文件 |
| 10 | 定标 | 确定中标供应商 |
| 11 | 中标/未中标通知 | 通知供应商结果 |
| 12 | 合同审批 | 合同签订前审批 |
| 13 | 生产 | 供应商生产制造 |
| 14 | 运输 | 货物运输到指定地点 |
| 15 | 验收 | 最终验收确认 |

## 五、业务规则

### 5.1 任务类型与节点必填关系

| 任务类型 | 必填节点数 | 必填节点 |
|---------|-----------|---------|
| 框架招标 | 12个 | 需求确认、供应商开发、技术交流、招标方案审批、发标、答疑、供应商回标、开标、定标、中标通知、合同审批、打样 |
| 单次采购≥100万 | 12个 | 同上 |
| 单一来源 | 4个 | 需求确认、发标、定标、合同审批 |
| 单次采购＜100万 | 4个 | 同上 |

### 5.2 项目编号规则

- 纵腾（ZT）：CG-2026-001（流水号按年递增）
- GOFO：GFCG-2026-001

### 5.3 项目自动完成

全部 15 个节点都设置了实际完成日期后，项目状态自动变为「项目完成」。

## 六、常见问题

**Q：登录后看到空白页面？**
A：检查浏览器控制台是否有报错，确认 VITE_API_URL 环境变量正确指向后端地址。

**Q：创建项目提示"项目名称已存在"？**
A：系统不允许同名项目，请更换名称。

**Q：Bot 不响应群聊消息？**
A：确保已@机器人，或在@后的5分钟内发送消息。超过5分钟需要重新@。

**Q：Bot 提示"仅负责人可操作"？**
A：只有项目负责人才能执行创建、更新、推进节点等写操作。非负责人只能查询。

**Q：节点状态显示不正确？**
A：节点状态根据日期自动计算——在计划周期内显示"进行中"，超过计划结束日期未完成显示"逾期"，有实际完成日期显示"已完成"。

**Q：如何解除群聊与项目的绑定？**
A：在群聊中发送"解绑"即可。解绑后可重新绑定其他项目。

**Q：创建项目时提示"项目名称已存在"？**
A：系统不允许同名项目，确认卡片点击"确认创建"时会自动检查。请更换项目名称后重新创建。
