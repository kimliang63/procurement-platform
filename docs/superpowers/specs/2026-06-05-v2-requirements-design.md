# 采购项目进度管理系统 — V2 需求设计

## 概述

在 V1 基础上全面升级为「采购项目进度管理系统」，核心变化：
- Dashboard 新增 BU/负责人/金额维度统计
- 项目列表从表格改为时间线视图
- 新增权限管理（采购员只看自己的项目）
- 新增业务规则引擎（快速规则 vs 招标规则）
- Bot 增强（群聊绑定、周报推送）

## 一、用户与角色

### 角色定义
- **管理员** — 看所有项目，管理用户权限
- **项目负责人（采购员）** — 只能看自己负责的项目，可编辑

### 公司与 BU
- 公司从飞书授权自动判断，无需手动选择
- 纵腾（ZT）：LBU / FBU / HQU / ABU / PBU
- GOFO：GUS / GUE

## 二、业务场景

### 场景 1：创建项目
1. 负责人在 Web 端或 Bot 私聊中创建项目
2. 系统根据公司自动选编号前缀，流水号自动生成（CG-2026-001 / GFCG-2026-001）
3. 负责人手动选择任务类型（单次＜100万 / 单次≥100万 / 单一来源 / 框架招标）
4. 系统自动初始化 16 个阶段节点

### 场景 2：项目进度跟踪
- Dashboard 展示统计卡片 + 按 BU / 按负责人维度聚合
- 项目列表内嵌横向时间线，点击节点弹窗编辑
- 节点状态自动计算：未开始 / 进行中 / 已完成 / 阻塞

### 场景 3：问题管理
- 问题关联项目和阶段，复用现有 issues 表
- 节点卡片内可直接添加该节点的问题

### 场景 4：飞书 Bot 交互
- 群聊绑定项目，负责人读写，非负责人只读
- 看板卡片展示进度条 + 问题统计
- 群聊指令触发推送到其他群

### 场景 5：项目完成
- 全部 16 个节点都有实际完成日期时，项目状态自动变为「项目完成」

## 三、功能清单

### 3.1 Web 平台

#### 3.1.1 左侧导航
- 项目看板 — Dashboard 统计 + 创建项目入口
- 项目列表 — 列表页内嵌时间线视图
- 问题跟踪 — 问题列表 + 筛选
- 授权管理 — 管理员给采购员开权限

#### 3.1.2 Dashboard 项目看板
- 统计卡片：项目总数 / 进行中 / 已定标（定标节点完成）/ 100万元以上项目数量
- 按 BU 统计：进行中数量 + 本年累计数量 + 年度累计采购金额及百分比
- 按负责人统计：进行中数量 + 本年累计数量
- 最右侧：创建项目快捷入口

#### 3.1.3 项目列表（时间线视图）
- 左上角：项目名称、项目编号
- 右上角：项目负责人、项目状态、X 个问题
- 主轴：16 个采购节点横向排列
- 节点颜色：已完成=绿色 / 当前节点=蓝色高亮 / 未开始=灰色 / 阻塞=红色
- 点击节点 → 弹窗编辑（计划日期、实际日期、状态、问题）
- 取消百分比进度条

#### 3.1.4 创建项目
字段：
- 项目名称 — 文本，必填
- 项目编号 — 自动生成（CG-2026-001 / GFCG-2026-001）
- 所属 BU — 下拉选择，根据公司自动筛选可选 BU
- 申请单号 — 文本，手动输入飞书 OA 单号
- 采购金额（万元）— 数字
- 负责人 — 从用户表选择
- 任务类型 — 下拉：单次采购＜100万 / 单次采购≥100万 / 单一来源 / 框架招标
- 项目状态 — 进行中 / 项目取消 / 项目暂停 / 项目完成
- 备注 — 多行文本

#### 3.1.5 项目状态
- 进行中 — 默认状态
- 项目完成 — 全部 16 个节点都有实际完成日期时自动更新
- 项目暂停 — 负责人手动设置
- 项目取消 — 负责人手动设置

#### 3.1.6 节点管理
16 个阶段节点：
需求确认 → 供应商开发 → 技术交流 → 打样 → 招标方案审批 → 发标 → 答疑 → 供应商回标 → 开标 → 定标 → 中标/未中标通知 → 合同审批 → 生产 → 运输 → 验收

节点卡片字段：
- 计划日期
- 实际日期
- 节点状态：未开始 / 进行中 / 已完成 / 阻塞
- 节点问题 — 复用 issues 表，通过 stage_key 关联
- 允许用户手工编辑所有字段

### 3.2 飞书 Bot

#### 3.2.1 群聊绑定
- 一个群聊只能关联一个项目
- @bot "绑定 项目名" 触发绑定，校验负责人身份
- 群聊中只允许操作当前绑定的项目

#### 3.2.2 权限控制
- 群聊 + 负责人：读写均可
- 群聊 + 非负责人：只允许查询，写操作提示「仅负责人可操作」
- 私聊：仅可操作自己负责的项目

#### 3.2.3 看板卡片
- 展示 13 阶段进度条 + 问题统计 + 预算 + 计划周期
- 按钮：查看Web端 / 更新节点 / 创建问题
- 更新节点、创建问题按钮仅负责人可点

#### 3.2.4 Bot 指令
- 群聊指令：查询进度、更新节点、创建问题、更新问题状态、推送进度到其他群
- 私聊：查询自己负责的项目、创建项目、创建后可选建群

#### 3.2.5 Bot 周报
- **管理员周报**：每周五汇总整体情况，向管理员及特定人推送
- **项目群周报**：每周五在机器人所在的项目群，向群内所有人推送本周的项目变化情况

## 四、规则库

### 4.1 项目编号规则
- 纵腾：CG-2026-001（最后三位流水号，后台自动编号）
- GOFO：GFCG-2026-001（最后三位流水号，后台自动编号）

### 4.2 任务分类规则

#### 快速规则（单次＜100万 / 单一来源）
必填节点（4 个）：
- 需求确认
- 发标
- 定标
- 合同签订

其余节点显示但可不填写。

#### 招标规则（单次≥100万 / 框架招标）
必填节点（12 个）：除以下 4 个外的所有节点
可不填：打样、生产、运输、验收

### 4.3 年度统计
- 年度累计采购金额 — 按自然年（1-12月）统计所有项目的预算总和
