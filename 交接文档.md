> 最后更新：2026-07-07

## 一、项目概览

供应链采购协同平台，连接采购方与供应商，覆盖采购全流程。

| 项 | 说明 |
|---|------|
| 项目名称 | 供应链采购协同平台（Procurement Platform） |
| 生产地址 | **https://hras-ai-asrm.ztn.cn/** |
| 代码仓库 | https://github.com/kimliang63/procurement-platform（接手后迁移到自己的 GitHub） |
| 技术栈 | React 18 + Ant Design 5 + Tailwind CSS 4 + Express 4 + MySQL 8.0 |
| 部署方式 | Docker 容器运行在内网服务器，HRAS nginx 反向代理到公网 |
| 数据库 | MySQL `assc_srm`，内网 10.100.12.119:63307 |
| 飞书 Bot | 应用名「采购项目小助手」，App ID: `cli_aa9c0ff0f8b99cc0` |

### 采购流程（15 阶段）

需求确认 → 供应商开发 → 技术交流 → 打样 → 招标方案审批 → 发标 → 答疑 → 供应商回标 → 开标 → 定标 → 中标/未中标通知 → 合同审批 → 生产 → 运输 → 验收

### 核心功能

| 模块 | 说明 |
|------|------|
| Dashboard 看板 | 项目统计 + BU/负责人图表 + 任务类型饼图 |
| 项目列表 | 15 节点时间线视图，节点状态颜色标记 |
| 项目详情 | 节点表格 + 问题列表，支持推进/异常标记 |
| 问题追踪 | 项目问题 CRUD，按优先级/状态筛选 |
| 用户管理 | 角色管理（admin/pm），仅管理员可见 |
| 飞书 Bot | 群聊绑定、自然语言交互（DeepSeek LLM）、周报推送 |
| 业务规则引擎 | 三维矩阵（采购方式 × 预算 × 是否单一来源）决定节点可见性 |

## 二、部署架构

### 2.1 拓扑

```
用户浏览器
    │
    └─ https://hras-ai-asrm.ztn.cn/（HRAS 壳子 nginx）
        │
        ├─ / → 前端静态文件
        ├─ /api/* → 后端 API
        ├─ /health → 健康检查
        └─ /webhook/bot → 飞书 Bot 回调
            │
            └─ 反向代理到内网 Docker 容器（10.100.x.x:3000）
                │
                ├─ MySQL (10.100.12.119:63307)
                ├─ 飞书开放平台 API（OAuth + Bot 消息）
                └─ DeepSeek LLM API（Bot 意图识别）
```

> **关键点**：前后端**同源**，Docker 镜像包含前端构建产物，Express 同时托管前端页面和 API。无需 CORS 配置，无需单独部署前端。

### 2.2 各组件地址

| 组件 | 地址 |
|------|------|
| **前端 + 后端** | https://hras-ai-asrm.ztn.cn/ |
| HRAS 壳子管理后台 | http://10.100.5.118:8066 |
| MySQL 数据库 | 10.100.12.119:63307（`assc_srm`） |
| 飞书开放平台 | https://open.feishu.cn（应用「采购项目小助手」） |
| DeepSeek | https://platform.deepseek.com |

### 2.3 Docker 部署

```bash
# 构建并启动
docker compose up -d

# 查看日志
docker compose logs -f

# 重启
docker compose restart

# 更新代码
git pull origin main
docker compose down
docker compose up -d --build
```

Dockerfile 多阶段构建：Stage 1 构建前端 → Stage 2 Node 22 Alpine 运行 Express，同时托管前端静态文件和 API。

### 3.2 关键设计

- **MCP 共享内核**：REST API 和飞书 Bot 调用同一套 MCP 函数，保证业务逻辑一致
- **前后端同源**：Docker 镜像包含前端构建产物，使用相对路径 `/api`，无需跨域
- **Bot 异步处理**：Webhook 立即返回 200，业务逻辑 fire-and-forget（8 秒超时兜底）
- **权限模型**：admin 看全部项目，pm 只看自己负责的项目

### 3.3 业务规则引擎

节点可见性由三维矩阵决定（`server/src/mcp/rules.js`）：

| 维度 | 取值 |
|------|------|
| 是否单一来源 | 是 / 否 |
| 预算 | <100万 / ≥100万 |
| 采购方式 | 框架类 / 项目类 |

每个节点取值 `required` / `visible` / `hidden`。快速规则（<100万/单一来源）仅 4 个必填节点，招标规则（≥100万/框架招标）12 个必填节点。

## 五、接手步骤

### 5.1 准备

| 账号 | 用途 |
|------|------|
| GitHub | Fork 原仓库继续开发 |
| 飞书开放平台 | 接管现有应用「采购项目小助手」（原管理员转让） |
| DeepSeek | Bot 意图识别 API Key |
| MySQL | 数据库（续用现有或自建） |
| Docker 服务器 | 部署后端服务 |

### 5.2 获取代码

```bash
git clone https://github.com/<你的GitHub用户名>/procurement-platform.git
cd procurement-platform
```

### 5.3 接管飞书应用

1. 联系原管理员在飞书开放平台 → 应用管理 → 成员管理 将你添加为**管理员**
2. 确认权限已开启（已配置好）：
   - `im:message:send_as_bot`
   - `im:chat:readonly`
   - `contact:user.base:readonly`
   - `authen:user.info:read`
3. 确认以下飞书开放平台配置：

| 配置项 | 填写 |
|--------|------|
| 事件订阅 → Request URL | `https://hras-ai-asrm.ztn.cn/webhook/bot` |
| 安全设置 → 重定向 URL | `https://hras-ai-asrm.ztn.cn/api/auth/feishu/callback` |
| 网页应用 → 主页 | `https://hras-ai-asrm.ztn.cn/` |

> App ID / App Secret 沿用现有值，无需重新创建。

### 5.4 获取 DeepSeek API Key

登录 https://platform.deepseek.com 创建 Key，确保账户有余额。

### 5.5 准备数据库

**续用现有**：联系管理员获取 `10.100.12.119:63307` 的访问权限。

**自建**：
```bash
docker run -d --name mysql -e MYSQL_ROOT_PASSWORD=<密码> -e MYSQL_DATABASE=assc_srm -p 3306:3306 mysql:8.0
mysql -h 127.0.0.1 -u root -p assc_srm < server/src/db-schema.sql
```

### 5.6 配置环境变量

```bash
cp .env.docker.example .env.docker
```

编辑 `.env.docker`：

```bash
PORT=3000

# MySQL
MYSQL_HOST=10.100.12.119
MYSQL_PORT=63307
MYSQL_USER=dev_hars_ai_user
MYSQL_PASSWORD=***
MYSQL_DATABASE=assc_srm

# 飞书应用（沿用现有凭证）
FEISHU_APP_ID=cli_aa9c0ff0f8b99cc0
FEISHU_APP_SECRET=***
FEISHU_VERIFY_TOKEN=***

# DeepSeek LLM
LLM_MODEL=deepseek-v4-flash
LLM_API_KEY=<你的DeepSeek API Key>
LLM_BASE_URL=https://api.deepseek.com

# 服务地址（前后端同源，全部填同一个域名）
SERVER_URL=https://hras-ai-asrm.ztn.cn
WEB_URL=https://hras-ai-asrm.ztn.cn
CORS_ORIGIN=https://hras-ai-asrm.ztn.cn

# HRAS 壳子
HRAS_SHELL_URL=http://10.100.5.118:8066
HRAS_SHELL_REGISTER_ENABLED=true
HRAS_MODULE_KEY=hras-procurement
HRAS_MODULE_NAME=采购项目进度管理
HRAS_MODULE_FRONTEND_URL=https://hras-ai-asrm.ztn.cn/
HRAS_MODULE_BACKEND_URL=https://hras-ai-asrm.ztn.cn/
```

### 5.7 启动

```bash
docker compose up -d
```

验证：

```bash
curl https://hras-ai-asrm.ztn.cn/health       # → {"status":"UP"}
curl https://hras-ai-asrm.ztn.cn/api/health    # → {"status":"ok"}
```

## 八、常见问题

### Bot 不回复
1. `curl https://hras-ai-asrm.ztn.cn/health`
2. 飞书开放平台 → 事件订阅 Request URL 是否为 `https://hras-ai-asrm.ztn.cn/webhook/bot`
3. DeepSeek API Key 是否有效
4. `docker compose logs -f | grep -E "Error|Webhook"`

### 登录失败
1. 飞书开放平台 → 重定向 URL 是否包含 `https://hras-ai-asrm.ztn.cn/api/auth/feishu/callback`
2. `SERVER_URL` 是否正确

### 用户角色不生效
登出重新飞书登录即可（OAuth 回调会清除缓存）。
