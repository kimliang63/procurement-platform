## 四、飞书应用配置

### 4.1 需要的权限范围

在飞书开放平台 → 应用 → 权限管理 中确认以下权限已开启：

| 权限 | 用途 |
|------|------|
| `bitable:app` | 多维表格读写 |
| `im:message:send_as_bot` | 发送消息 |
| `im:chat:readonly` | 读取群聊信息 |
| `contact:user.base:readonly` | 读取用户基本信息 |
| `authen:user.info:read` | OAuth 登录获取用户信息 |

### 4.2 事件订阅

在飞书开放平台 → 应用 → 事件订阅 中确认：

| 事件 | Webhook 地址 |
|------|-------------|
| `im.message.receive_v1` | https://procurement-server-production-b325.up.railway.app/webhook/bot |

## 七、部署流程

### 7.1 前端部署（Vercel）
```bash
cd web
npx vercel --prod --yes
```
或推送代码到 main 分支后 Vercel 自动部署。

### 7.2 后端部署（Railway）
```bash
npx railway up --service procurement-server
```
或推送代码到 main 分支后 Railway 自动部署（需确认已配置 GitHub 集成）。

### 7.3 本地开发
```bash
# 后端
cd server
cp .env.example .env  # 填入环境变量
npm install
npm run dev  # 端口 4000

# 前端
cd web
npm install
npm run dev  # 端口 5173
```

## 八、常见维护操作

### 8.1 添加新用户
用户通过飞书 OAuth 登录后自动注册，无需手动添加。

### 8.2 修改用户角色
在 Web 端 → 用户管理（仅管理员可见）中修改。

### 8.3 查看 Bot 日志
在 Railway Dashboard → procurement-server → Logs 中查看。

### 8.4 回滚版本
```bash
# 前端回滚
git checkout <commit-sha> -- web/
cd web && npx vercel --prod --yes

# 后端回滚
git checkout <commit-sha> -- server/
npx railway up --service procurement-server
```

## 九、注意事项

1. **飞书 Bitable 是无模式数据库**，修改表结构（增删字段）不会影响现有数据
2. **Bot 的 LLM 依赖 DeepSeek**，需确保账户余额充足
3. **Vercel 和 Railway 都有免费额度限制**，超出后可能产生费用
4. **飞书应用的权限变更需要重新发布**，在开放平台提交审核
5. **环境变量修改后需要重启服务**（Railway 会自动重启）
