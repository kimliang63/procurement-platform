# 采购协同平台 — 用户测试指引

## 测试目标

验证 Bot 对话、项目管理、节点推进等功能是否满足日常采购工作需要。

## 测试前准备

1. 确保已加入飞书测试群（或私聊 Bot）
2. 打开 Web 端：`http://localhost:5173`（开发环境）
3. 准备一个测试项目名称，如"UAT测试项目-你的名字"

## 测试方式

在飞书中给 Bot 发消息，按下面的场景逐项操作，每完成一项在"通过"列打 ✅。

---

## 一、创建项目（Bot 对话）

### 1.1 发起创建

在私聊或群聊中 @机器人，依次发送以下消息，观察 Bot 是否正确追问：

| 输入 | Bot 应该回复 | 通过 |
|------|------------|------|
| 创建一个新项目 | 追问项目名称、品类等信息 | |
| 创建项目UAT测试-你的名字 | 追问品类、部门、预算、日期（名称已记住） | |
| 创建项目UAT测试-你的名字，品类设备 | 追问部门、预算、日期（名称+品类已记住） | |

### 1.2 补充信息

继续发送以下消息，观察 Bot 是否累积已填信息：

| 输入 | Bot 应该回复 | 通过 |
|------|------------|------|
| 设备 | 品类已记录，追问下一个字段 | |
| FBU | 部门已记录，追问下一个字段 | |
| 预算80万 | 预算已记录，追问下一个字段 | |
| 开始2026-07-01，结束2026-12-31 | 日期已记录 | |
| 负责人是我 | 信息完整，发送确认卡片 | |

### 1.3 重名校验

| 输入 | Bot 应该回复 | 通过 |
|------|------------|------|
| 创建项目UAT测试-你的名字（用刚才已创建的名称） | 提示"项目名称已存在，请换个名称" | |

### 1.4 确认创建

收到确认卡片后：

| 操作 | Bot 应该回复 | 通过 |
|------|------------|------|
| 点击"确认创建"按钮 | 项目创建成功，显示项目编号 | |
| 或者发送"确认"/"好的"/"可以"/"没问题" | 同上 | |

### 1.5 取消创建

重新发起创建流程，收到确认卡片后：

| 操作 | Bot 应该回复 | 通过 |
|------|------------|------|
| 点击"取消"按钮 | 已取消创建 | |
| 或者发送"取消"/"不要"/"算了" | 同上 | |

### 1.6 防重复点击

| 操作 | Bot 应该回复 | 通过 |
|------|------------|------|
| 点击"确认创建"后立即再点一次 | 第二次显示"正在处理中，请勿重复点击" | |

## 三、节点管理（Bot 对话）

先确保已创建一个项目并处于"需求确认"阶段。

| 输入 | Bot 应该回复 | 通过 |

## 问题反馈模板

发现问题时请按以下格式记录：

```
【场景】：创建项目时输入重名
【操作】：发送"创建项目ABC"（ABC已存在）
【预期】：提示名称已存在
【实际】：没有提示，直接追问品类
【严重程度】：P0/P1/P2
```

- **P0**：阻塞核心流程，无法继续使用
- **P1**：功能异常但有绕过方案
- **P2**：体验问题，不影响功能
