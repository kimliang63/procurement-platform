# 采购协同平台 — 用户测试指引

## 测试目标

以一个完整的采购项目为线索，走通"Bot 创建项目 → Web 查看管理 → 节点推进 → 问题跟踪"全流程。

| 端 | 地址 |
|---|------|
| Web 端 | https://kimliang63.github.io/procurement-platform/ |
| Bot | 飞书私聊或测试群中 @采购协同助手 |

## 完整流程（约 10 分钟）

跟着下面的步骤走一遍，遇到问题记录在最后的反馈表里。

### 第 1 步：通过 Bot 创建项目

打开飞书，私聊或在测试群里 @机器人，发送：

> 创建一个新项目

Bot 会依次追问项目名称、品类、部门、预算、计划周期。你逐条回复即可，比如：

> UAT测试项目-你的名字

> 设备

> FBU

> 预算80万

> 开始2026-07-01，结束2026-12-31

> 负责人是我

信息补全后 Bot 会发一张确认卡片，点击 **"确认创建"**（或回复"确认"/"好的"/"可以"）。

**留意：**
- 如果名称和已有项目重复，Bot 会提示"项目名称已存在，请换个名称"
- 点击确认后，不要重复点击，等 Bot 回复创建成功

### 第 2 步：在 Web 端查看项目

1. 打开 https://kimliang63.github.io/procurement-platform/
2. 点击 **飞书登录**，完成授权
3. 在项目列表中找到刚才创建的项目，点进去查看详情
4. 确认：名称、品类、负责人、部门、预算、计划周期都正确

### 第 3 步：通过 Bot 推进节点

回到飞书，给 Bot 发消息：

> 查看项目节点

Bot 会列出 13 个节点的当前状态。然后发送：

> 需求确认完成了

Bot 回复"需求确认已完成，下一步：供应商开发"。

### 第 4 步：在 Web 端验证节点状态

回到 Web 端项目详情页，刷新页面，确认"需求确认"节点已标记为完成。

### 第 5 步：通过 Bot 标记异常（可选）

回到飞书，发送：

> 供应商开发有问题

Bot 回复"供应商开发已标记为异常"。在 Web 端刷新确认状态同步。

### 第 6 步：通过 Web 端推进节点

在 Web 端项目详情页：
1. 选择"技术交流"节点
2. 填写 **计划完成日期**
3. 填写 **实际完成日期**
4. 点击保存 → 节点自动标记为完成

## 补充验证（可选）

如果时间充裕，可以额外试试：

**Bot 对话：**
- 发送"查看项目" → 应列出所有项目
- 发送项目名称（如"UAT测试项目-你的名字"）→ 应显示项目详情
- 在群里 @机器人 后，5 分钟内不@直接发消息 → 应自动响应
- 发送乱码如"asdfghjkl" → 应返回引导提示

**Web 端：**
- 未登录时直接访问项目页 → 应跳转到登录页
- 节点没有填写计划完成日期时尝试完成 → 应提示必填
